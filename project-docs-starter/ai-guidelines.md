# AI Engineering Guidelines

## Core Principles

- Current codebase is the source of truth
- Prefer incremental improvement over rewrites
- Keep implementation scope small
- Avoid unnecessary architectural expansion

---

## Context Rules

Only read:
- relevant spec docs
- relevant architecture docs
- target files
- current-state.md

Avoid:
- rereading all historical logs
- loading unrelated documents
- scanning the full repo unnecessarily

---

## Task Workflow

1. Read relevant context
2. Produce spec/plan first
3. Wait for approval
4. Implement one task only
5. Run verification
6. Update documentation
7. Stop for review
